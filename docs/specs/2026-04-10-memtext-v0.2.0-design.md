# Memtext v0.2.0 Design - SQLite + Cross-Project Context

## Overview
Add SQLite metadata layer and cross-project context management to memtext for AI agents.

## Architecture

### Directory Structure
```
.context/
├── memtext.db          # SQLite database
├── identity.md        # Project identity
├── decisions.md      # Architecture decisions  
└── session-logs/    # Session notes
    └── YYYY-MM-DD.md
```

### Database Schema

```sql
-- Context entries with metadata
CREATE TABLE context_entries (
    id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,           -- Entry title/headline
    content TEXT NOT NULL,         -- Full markdown content
    entry_type TEXT NOT NULL,          -- decision, pattern, note, error, convention
    importance INTEGER DEFAULT 1,  -- 1-5 scale
    tags TEXT,                    -- comma-separated
    source TEXT DEFAULT 'manual',   -- session, auto, manual
    linked_files TEXT,             -- comma-separated file paths
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_accessed TIMESTAMP,
    access_count INTEGER DEFAULT 0
);

-- Projects with .context/ folders (stored centrally)
CREATE TABLE projects (
    id INTEGER PRIMARY KEY,
    path TEXT UNIQUE NOT NULL,
    name TEXT,
    last_active TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Full-text search (rebuilt on changes)
CREATE VIRTUAL TABLE context_fts USING fts5(title, content, entry_type, tags);
```

### Project Registry Location
Central registry stored at: `~/.config/memtext/projects.db` (or `%APPDATA%/memtext/projects.db` on Windows)

## Agent Skills

### context-manager
- Add/update/delete context entries
- Auto-tag based on content analysis
- Link related files

### context-pruner
- Identify stale entries (not accessed in N days)
- Find overlapping/redundant context
- Suggest entries for removal

### context-retriever
- Query by type, tags, importance, date
- Full-text search across projects
- Sort by relevance

### context-linker
- Auto-detect related context pieces
- Suggest links based on file changes
- Track context relationships

## CLI Commands

| Command | Description |
|---------|-------------|
| `memtext init` | Initialize with SQLite |
| `memtext add <text> --type decision --tags x,y` | Add context entry |
| `memtext query <text> --type pattern --importance 3+` | Search context |
| `memtext list --type decision --limit 10` | List entries |
| `memtext prune --days 30` | Find stale entries |
| `memtext link <entry-id> <file>` | Link file to entry |
| `memtext projects` | List projects on drive |
| `memtext sync` | Sync .context/ index |

## Cross-Project Features

- Scan drive for `.context/` folders on init
- Store project registry in central SQLite (or config)
- Query context across projects
- Export/import context between projects
- Track project relationships

## Migration (v0.1.x → v0.2.0)
- `memtext migrate` command converts existing markdown files to SQLite entries
- Parses identity.md, decisions.md into structured entries
- Maps session-logs/*.md to entries with type=note

## Export/Import Format
- **JSON**: Full fidelity, includes all metadata
- **Markdown**: Human-readable, loses metadata but portable

## Cross-Project Scanning
- Lazy scanning: `memtext projects` scans on demand
- Background sync with `--scan` flag for full drive scan
- Projects added explicitly via `memtext projects add <path>`

## Success Criteria

1. SQLite database created on `memtext init`
2. All new CLI commands implemented
3. Agent skills accessible via CLI or import
4. Cross-project registry works
5. Migration command converts v0.1.x data
6. Export/import functionality works
